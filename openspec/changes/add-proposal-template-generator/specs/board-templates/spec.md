## ADDED Requirements

### Requirement: Generate Template from Text Endpoint
The system SHALL expose `POST /api/templates/generate` as an authenticated endpoint that accepts unstructured text and uses AI to generate a structured board template. The request body SHALL include `content` (required, string, max 50000 characters — the raw text to parse), `title` (optional, string, max 200 — override for the template title), `description` (optional, string, max 1000), `category_id` (optional, UUID), and `visibility` (optional, default `private`). The endpoint SHALL: (1) send the content to the AI template generation node, (2) receive a structured DAG of tasks with dependencies and subtasks, (3) persist the result as a BoardTemplate with associated TemplateTasks, TemplateTaskDependencies, and TemplateSubtasks in a single transaction, (4) return the created template with status 201. If the AI cannot extract meaningful tasks from the content, the endpoint SHALL return 422 with an error message.

#### Scenario: Generate template from pasted text
- **WHEN** an authenticated user sends `POST /api/templates/generate` with `{"content": "Step 1: Research destinations\nStep 2: Book flights\nStep 3: Find accommodation\nStep 4: Pack bags\nStep 5: Travel", "title": "Trip Planning"}`
- **THEN** a template is created with tasks extracted from the text, dependencies inferred by the AI, and a goal node
- **AND** the response status is 201 with the full template details

#### Scenario: Generate template from detailed project brief
- **WHEN** an authenticated user sends `POST /api/templates/generate` with a 2000-character project brief describing a software launch
- **THEN** the AI extracts tasks, identifies parallel work streams, creates convergence nodes, and produces a valid DAG template

#### Scenario: Generate template from content with no actionable tasks
- **WHEN** a user sends `POST /api/templates/generate` with `{"content": "The weather is nice today."}`
- **THEN** the response status is 422 with an error indicating no actionable tasks could be extracted

#### Scenario: Generate template with empty content rejected
- **WHEN** a user sends `POST /api/templates/generate` with `{"content": ""}`
- **THEN** the response status is 422 with a validation error

### Requirement: Generate Template from Document Endpoint
The system SHALL expose `POST /api/templates/generate-from-document` as an authenticated endpoint that accepts a file upload and uses AI to generate a structured board template. The endpoint SHALL accept multipart form data with fields: `file` (required, uploaded file), `title` (optional, string), `description` (optional, string), `category_id` (optional, UUID), `visibility` (optional, default `private`). Supported file types SHALL be: `.txt`, `.md`, `.pdf`, `.docx`. Maximum file size SHALL be 5MB. The endpoint SHALL: (1) extract text content from the uploaded file, (2) delegate to the same AI template generation pipeline as the text endpoint, (3) persist and return the created template with status 201.

#### Scenario: Generate template from PDF document
- **WHEN** an authenticated user uploads a PDF project plan via `POST /api/templates/generate-from-document`
- **THEN** text is extracted from the PDF, tasks are generated by AI, and a template is created
- **AND** the response status is 201

#### Scenario: Generate template from Markdown file
- **WHEN** a user uploads a `.md` file containing a checklist with `- [ ]` items
- **THEN** the AI parses the checklist structure and generates a template with corresponding tasks

#### Scenario: Unsupported file type rejected
- **WHEN** a user uploads a `.xlsx` file
- **THEN** the response status is 422 with an error indicating the file type is not supported

#### Scenario: File too large rejected
- **WHEN** a user uploads a 10MB PDF file
- **THEN** the response status is 422 with an error indicating the file exceeds the 5MB limit

### Requirement: Template Generation UI — Text Input
The system SHALL provide a "Generate from Text" option on the templates page. The UI SHALL include: a large text area for pasting or typing content (with placeholder text suggesting formats like checklists, project briefs, or step-by-step plans), a "Generate Template" button, and a loading state while the AI processes the content. After generation, the user SHALL be shown a preview of the generated template with an edit form for metadata (title, description, category, visibility) pre-filled from AI suggestions, and a "Save" button to confirm.

#### Scenario: User generates template from pasted text
- **WHEN** a user clicks "Generate from Text", pastes a project outline, and clicks "Generate Template"
- **THEN** a loading indicator appears while the AI processes the content
- **AND** upon completion, a preview of the generated template is displayed with editable metadata fields

#### Scenario: User edits generated template before saving
- **WHEN** the AI generates a template and the user modifies the title and selects a category
- **THEN** the saved template reflects the user's edits

### Requirement: Template Generation UI — Document Upload
The system SHALL provide a "Upload Document" option alongside the text input on the template generation flow. The UI SHALL include: a file drop zone or file picker accepting `.txt`, `.md`, `.pdf`, `.docx` files (max 5MB), file type and size validation with user-friendly error messages, and the same generation flow (loading → preview → edit metadata → save) as the text input path.

#### Scenario: User uploads a document to generate a template
- **WHEN** a user drags a PDF file onto the upload zone and confirms
- **THEN** the file is uploaded, processed by the AI, and a template preview is displayed

#### Scenario: User sees error for invalid file type
- **WHEN** a user tries to upload a `.png` file
- **THEN** the UI shows an error message indicating only .txt, .md, .pdf, and .docx files are supported
